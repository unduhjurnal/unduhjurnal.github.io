1. 1. Jelaskan perbedaan aplikasi berbasis web dan aplikasi berbasis mobile serta jelaskan bagaimana sebuah perusahaan menentukan kapan harus menggunakan aplikasi berbasis web dan kapan harus menggunakan aplikasi berbasis mobile!


Aplikasi berbasis web dan aplikasi berbasis mobile memiliki perbedaan dalam hal platform, aksesibilitas, pengembangan, dan pengalaman pengguna. Berikut adalah penjelasan tentang perbedaan antara keduanya:

1. Platform:

- Aplikasi berbasis web dirancang untuk diakses melalui browser web dan dapat dijalankan di berbagai platform, seperti desktop, laptop, tablet, dan perangkat seluler.

- Aplikasi berbasis mobile dirancang khusus untuk dijalankan pada perangkat seluler seperti smartphone atau tablet.

2. Aksesibilitas:

- Aplikasi berbasis web dapat diakses dengan mudah melalui URL dan tidak memerlukan instalasi tambahan pada perangkat pengguna. Pengguna hanya perlu memiliki browser web yang kompatibel.

- Aplikasi berbasis mobile harus diunduh dan diinstal secara khusus dari toko aplikasi (misalnya, App Store untuk iOS atau Google Play Store untuk Android) sebelum pengguna dapat menggunakannya.

3. Pengembangan:

- Aplikasi berbasis web biasanya dikembangkan menggunakan teknologi web seperti HTML, CSS, dan JavaScript. Pengembang dapat menggunakan kerangka kerja (framework) seperti React, Angular, atau Vue.js untuk mempermudah pengembangan.

- Aplikasi berbasis mobile memerlukan pengembangan menggunakan bahasa pemrograman yang spesifik untuk platform tertentu, seperti Swift atau Objective-C untuk iOS, dan Java atau Kotlin untuk Android.

4. Pengalaman Pengguna:

- Aplikasi berbasis web memiliki desain responsif yang dapat menyesuaikan tampilan dengan baik pada berbagai perangkat. Namun, pengalaman pengguna mungkin berbeda-beda tergantung pada browser yang digunakan dan kualitas koneksi internet.

- Aplikasi berbasis mobile dapat memberikan pengalaman pengguna yang lebih terintegrasi dengan perangkat seluler, termasuk akses ke fitur-fitur perangkat seperti kamera, GPS, notifikasi push, dan lainnya.

Keputusan perusahaan untuk menggunakan aplikasi berbasis web atau aplikasi berbasis mobile tergantung pada beberapa faktor, seperti:

1. Tujuan Aplikasi:

- Jika aplikasi bertujuan untuk menyediakan informasi umum yang mudah diakses, seperti situs web perusahaan, blog, atau platform e-commerce, aplikasi berbasis web mungkin lebih cocok.

- Jika aplikasi dirancang untuk memberikan pengalaman yang kaya dan terintegrasi dengan fitur-fitur perangkat seluler, seperti aplikasi perbankan, jejaring sosial, atau permainan mobile, maka aplikasi berbasis mobile lebih disarankan.

2. Target Pengguna:

- Jika target pengguna aplikasi adalah pengguna yang menggunakan berbagai platform dan perangkat, aplikasi berbasis web dapat mencakup lebih banyak pengguna potensial, karena dapat diakses dari berbagai perangkat.

- Jika target pengguna aplikasi adalah pengguna yang menggunakan perangkat seluler secara eksklusif, seperti aplikasi berbasis lokasi atau aplikasi berbasis kamera, maka aplikasi berbasis mobile dapat memberikan pengalaman yang lebih terarah dan terintegrasi.

3. Fitur dan Kebutuhan Teknis:

- Jika aplikasi memerl

ukan akses ke fitur-fitur perangkat seluler khusus, seperti GPS, kamera, sensor gerak, atau pemrosesan offline, maka aplikasi berbasis mobile menjadi pilihan yang lebih baik.

- Jika aplikasi hanya memerlukan tampilan dan interaksi dasar tanpa memerlukan akses ke fitur perangkat seluler, maka aplikasi berbasis web dapat memenuhi kebutuhan tersebut dengan lebih efisien.

Perusahaan perlu menganalisis kebutuhan, tujuan, dan karakteristik pengguna aplikasi mereka untuk menentukan apakah aplikasi berbasis web atau aplikasi berbasis mobile lebih cocok dalam konteks yang diberikan. Dalam beberapa kasus, perusahaan juga dapat mempertimbangkan untuk mengembangkan aplikasi hibrida yang menggabungkan kelebihan dari kedua platform tersebut.

Pengalaman Pengguna: Jika aplikasi perlu memberikan pengalaman yang dioptimalkan dan kustomisasi sesuai dengan perangkat mobile, seperti antarmuka sentuh dan notifikasi push, aplikasi berbasis mobile dapat memberikan pengalaman yang lebih baik.

Ketersediaan dan Aksesibilitas: Jika aplikasi harus dapat diakses dengan mudah tanpa memerlukan instalasi khusus, aplikasi berbasis web lebih praktis karena dapat diakses melalui browser.

3. Di bntu ya kak Apa itu pengembangan aplikasi berbasis web? Jelaskan arsitektur dasar aplikasi web dan sebutkan beberapa teknologi yang digunakan dalam pengembangan aplikasi berbasis web.


Jawaban:

Pengembangan aplikasi berbasis web adalah proses menciptakan dan mengembangkan aplikasi yang dapat diakses melalui web browser menggunakan jaringan internet. Aplikasi web ini berjalan di server dan dapat diakses oleh pengguna melalui peramban web mereka tanpa perlu menginstal aplikasi secara lokal di perangkat mereka.

Arsitektur dasar aplikasi web terdiri dari tiga komponen utama:

1. Front-End (Klien): Komponen front-end bertanggung jawab untuk interaksi langsung dengan pengguna. Ini adalah bagian yang tampak oleh pengguna melalui tampilan antarmuka (user interface/UI). Front-end terdiri dari bahasa markah (HTML), gaya (CSS), dan skrip (JavaScript). HTML digunakan untuk membuat struktur dan elemen halaman web. CSS digunakan untuk mengatur tampilan dan desain halaman web. JavaScript memberikan interaktivitas dan fungsionalitas pada aplikasi web.

2. Back-End (Server): Komponen back-end berada di sisi server dan bertanggung jawab untuk memproses permintaan dari klien, berinteraksi dengan database, dan melakukan logika bisnis yang diperlukan. Server back-end menghasilkan dan mengirimkan data yang diminta oleh klien. Bahasa pemrograman seperti JavaScript (Node.js), PHP, Python, Ruby, dan Java umum digunakan dalam pengembangan back-end.

3. Database: Database digunakan untuk menyimpan dan mengelola data yang diperlukan oleh aplikasi web. Data ini dapat berupa informasi pengguna, konten aplikasi, atau informasi lain yang relevan. Teknologi basis data seperti MySQL, Oracle, MongoDB, dan PostgreSQL digunakan untuk menyimpan dan mengatur data dalam aplikasi web.

Beberapa teknologi lain yang digunakan dalam pengembangan aplikasi web meliputi:

- Framework: Framework seperti React, Angular, dan Vue.js digunakan untuk mempercepat pengembangan aplikasi web dengan menyediakan alat dan struktur yang telah siap untuk digunakan.

- RESTful API: Representational State Transfer (REST) adalah pendekatan untuk membangun API (Application Programming Interface) yang dapat digunakan oleh aplikasi web untuk berkomunikasi dengan server dan mengambil atau memperbarui data.

- Pyton diketahui sebagai bahasa pemrograman yang serba bisa baik itu untuk pembuatan aplikasi desktop, mobile dan web.

- Cloud Computing: Layanan cloud seperti Amazon Web Services (AWS) dan Google Cloud Platform (GCP) digunakan untuk menyimpan dan mengelola aplikasi web di lingkungan cloud, yang menyediakan skalabilitas, ketersediaan, dan keamanan yang diperlukan.

- Masyarakat pembangunan API menggunakan spesifikasi OpenAPI dan pendekatan pembangunan API terbaik.

- Beberapa alat pengembangan web yang populer termasuk Git untuk kerjasama pengembangan kode, Docker untuk pengelolaan kontainer, dan Jenkins untuk otomatisasi pengembangan dan penyiapan.

5. Apa dan bagaimanakah cara mengatasi permasalahan sql injection pada aplikasi berbasis web?


Jawaban:

Cara menghindari SQL Injection :

> Validasi Input

Proses validasi ditujukan untuk memverifikasi apakah jenis input yang dikirimkan oleh pengguna diperbolehkan atau tidak. Validasi input memastikan itu adalah tipe, panjang, format, dan sebagainya yang diterima. Hanya nilai yang lolos validasi yang dapat diproses. Ini membantu menangkal setiap perintah yang dimasukkan dalam string input. Di satu sisi, itu mirip dengan mencari untuk melihat siapa yang mengetuk sebelum membuka pintu.

>Parametrisasi Query

Query yang di parameter adalah sarana untuk menyusun pernyataan SQL sehingga Anda kemudian dapat menyediakan parameter agar pernyataan dapat dijalankan. Metode ini memungkinkan database untuk mengenali kode dan membedakannya dari data input.

>Saved Procedure

mengharuskan pengembang untuk mengelompokkan satu atau beberapa pernyataan SQL ke dalam unit logis untuk membuat rencana eksekusi. Eksekusi berikutnya memungkinkan pernyataan secara otomatis di parameter. Sederhananya, ini adalah jenis kode yang dapat disimpan untuk nanti dan digunakan berkali-kali.

>Escaping

Selalu gunakan fungsi yang melarikan karakter untuk input yang disediakan pengguna yang disediakan oleh setiap sistem manajemen database (DBMS). Hal ini dilakukan untuk memastikan DBMS tidak pernah membingungkannya dengan pernyataan SQL yang diberikan oleh pengembang.

18. Cara pengaksesan database melalui aplikasi berbasis web


1. Client mengakses suatu halaman di web server (melakukan Request)

2. Jika file yang diakses adalah PHP, maka akan diteruskan ke PHP Interpreter untuk dieksekusi

3. Jika dalam file PHP ada function yang akan mengakses database, maka PHP interpreter akan mengakses

database sesuai script PHP yang ada.

4. Database server mengirimkan data (jika pencarian atau pengambilan data) atau informasi mengenai

aktivitas web yang dilakukan (status penyisipan, pengeditan dan penghapusan).

5. PHP Interperter mengembalikan hasil interpretasi PHP ke Web Server untuk dikirimkan ke client

6. Web Server mengirim hasil eksekusi file ke client (dalam bentuk halaman web) sebagai Response.

32. aplikasi control panel berbasis web yang awalnya dikembangkan oleh jamie cameron untuk melakukan administrasi sistem linux,kemudian berkembang menjadi sebuah web control panel yang mendukung hampir semua fitur manajemen sistem server adalah?


Jawaban:

Aplikasi control panel berbasis web yang awalnya dikembangkan oleh Jamie Cameron dan saat ini telah berkembang menjadi sebuah web control panel yang mendukung hampir semua fitur manajemen sistem server adalah Webmin.

Webmin adalah sebuah alat administrasi sistem berbasis web yang dirancang untuk melakukan administrasi pada sistem operasi seperti Linux, Unix, dan Windows melalui antarmuka web yang mudah digunakan. Dengan menggunakan Webmin, pengguna dapat melakukan manajemen dan konfigurasi server melalui browser web dari jarak jauh.

Fitur-fitur Webmin meliputi:

1. Administrasi jaringan: Meliputi pengelolaan antarmuka jaringan, pengaturan firewall, pengaturan DNS, pengaturan DHCP, pengelolaan VPN, dan lain-lain.

2. Pengelolaan server web: Memungkinkan pengguna mengelola server web seperti Apache, Nginx, dan Lighttpd. Pengguna dapat mengkonfigurasi situs web, mengelola virtual host, mengontrol modul dan ekstensi, serta mengelola izin akses file dan direktori.

3. Administrasi server basis data: Mendukung manajemen database seperti MySQL, PostgreSQL, dan MongoDB. Pengguna dapat membuat, menghapus, dan mengelola database serta user-nya, serta melakukan tugas administrasi lainnya.

4. Administrasi sistem file: Memungkinkan pengguna mengelola file dan direktori, mengubah izin akses, memindahkan atau menghapus file, serta melakukan tugas administrasi file system lainnya.

5. Pengelolaan pengguna dan grup: Memungkinkan pengguna untuk membuat, menghapus, dan mengelola pengguna dan grup, mengubah izin dan hak akses, serta melacak aktivitas pengguna.

6. Pemantauan sistem: Menyediakan informasi tentang penggunaan sumber daya sistem seperti penggunaan CPU, memori, dan ruang disk. Juga mendukung penjadwalan tugas dan pemberitahuan sistem.

Webmin memiliki antarmuka pengguna yang intuitif dan mudah digunakan, serta mendukung berbagai jenis server dan protokol. Dengan bekerja dengan Webmin, administrator sistem dapat dengan mudah mengelola server dan melakukan tugas administrasi sistem secara efisien melalui antarmuka web yang nyaman.

1. Perencanaan adalah membuat semua rencana yang berkaitan dengan proyek sistem informasi.

2. Setelah perencanaan selesai, langkah berikutnya adalah membuat analisa (analyst). Analisa adalah menganalisa workflow sistem informasi yang sedang berjalan dan mengindentifikasi apakah workflow telah efisien dan sesuai standar tertentu. Analisa dilakukan oleh Business Processs Analyst (BPA) yang berpengalaman dan/atau memahami workflow sistem manajemen di area yang sedang dianalisa.

6.Implementasi adalah proses untuk menerapkan sistem informasi yang telah dibangun agar user menggunakannya untuk menggantikan sistem informasi yang lama.

Semua aplikasi memiliki implementasi yang jelas sesuai dengan peruntukannya, maka dalam proses analisa sebelumnya kita memperhatikan pula, siapa yang akan mengimplementasikan atau menggunakan aplikasi ini kedepannya.

40. Contoh Proposal penelitian tentang Manajemen Informatika


SISTEM INFORMASI PERPUSTAKAAN  BERBASIS CLIENT SERVER PADA SMA NEGERI 1 BANJAR

BAB I PENDAHULUAN 

1.1.    Latar Belakang Masalah

Dewasa ini perkembangan teknologi informasi semakin pesat, sehingga banyak bidang perkerjaan yang dulunya dikerjakan manual oleh manusia sekarang bisa diganti oleh komputer. Dengan adanya sistem komputerisasi disegala bidang berbagai macam pekerjaan kini bisa diselesaikan dengan waktu yang cepat dan terstrukur.

Pada saat ini banyak perusahaan dan instansi pemerintah yang mulai melakukan komputerisasi pada setiap bidang pekerjaannya. Salah satu bidang yang kini banyak menggunakan  komputer adalah dalam bidang pengolah data. Oleh karena itu penulis akan mencoba membuat sebuah sistem informasi pengolahan data perpustakaan pada sebuah sekolah menengah umum yaitu SMA Negeri 1 Banjar.

Pada mulanya perpustakaan ini masih menggunakan cara pencatatan data dengan sangat sederhana yaitu hanya menggunakan aplikasi Microsoft Excel. Dengan cara pencatatan tersebut dalam pembuatan laporan masih belum memenuhi kebutuhan terutama untuk laporan data yang berdasarkan kriteria tertentu seperti laporan buku dengan penerbit tertentu, laporan buku yang dipinjam dan laporan buku perkelompok. Selain itu dalam perpustakaan juga tidak tersedia fasilitas katalog buku sehingga pengunjung perpustakaan yang akan meminjam buku harus mencari buku yang dipinjamnya secara manual.

Oleh sebab itu untuk meningkatkan kinerja dan pelayanan perpustakaan diperlukan suatu sistem yang mampu mengolah data baik itu data buku atau transaksi yang baik dan terstruktur serta mampu memberikan laporan yang akurat. Selain itu diperlukan juga sistem yang mampu menyedikan fasilitas katalog buku, sehingga pengunjung perpustakaan tinggal mencari buku yang akan dipinjamnya dari fasiltas katalog tersebut. Dari kebutuhan sistem informasi yang sangat kompleks tersebut diperlukan suatu sistem informasi yang mampu menangani kebutuhan tersebut secara sekaligus.

 

1.2.    Rumusan Masalah

Dari latar belakang masalah di atas dapat dirumuskan beberapa rumusan masalah yang akan dibahas dalam penelitian ini yaitu :

Bagaimana membuat suatu sistem informasi yang dapat melakukan pengolahan data anggota perpustakaan, data buku, denda transaksi peminjaman dan pengembalian buku serta petugas dengan cepat.Bagaimana membuat suatu informasi yang dapat memberikan laporan baik data anggota, data buku, denda dan data transaksi sesuai dengan kebutuhan.Bagaimana membuat suatu sistem informasi yang bisa diakses oleh banyak pihak yang membutuhkan dalam waktu yang bersamaan.

1.3.    Batasan Masalah

Dari rumusan masalah diatas penulis membatasi permasalah yang akan dibahas yaitu hanya mencakup :

Pembuatan suatu sistem informasi pengolahan data yang hanya mengolah  data anggota perpustakaan, petugas, data buku, transaksi peminjaman dan pengembalian buku serta dendaPembuatan laporan anggota, petugas, buku, peminjaman, pengembalian dan denda.Pembuatan suatu sistem informasi berbasis client server dalam sekala kecil.

 

1.4.    Tujuan Penelitian

Membuat suatu sistem informasi yang dapat melakukan pengolahan data anggota perpustakaan, data buku, denda transaksi peminjaman dan pengembalian buku serta petugas dengan cepat.Membuat suatu informasi yang dapat memberikan laporan baik data anggota, data buku, denda dan data transaksi sesuai dengan kebutuhan.Membuat suatu sistem informasi yang bisa diakses oleh banyak pihak yang membutuhkan dalam waktu yang bersamaan.

1.5.    Manfaat Penelitian

Dengan adanya kegiatan penelitian ini, diharapkan dapat memberikan manfaat bagi beberapa pihak antara lain :

Meningkatkan mutu pelayanan perpustakaan.Meningkatkan kinerja pengelola perpustakaan.

1.6.    Metode Penelitian

Metode Deskriptif

Metode yang pemecahan masalahnya dilakukan dengan cara  mendeskripsikan suatu fakta dalam arti yang luas menjadi lebih khusus.

Metode Eksperimen

Yaitu metode yang digunakan dengan cara melakukan pengujian atau percobaan terhadap data yang diolah pada sistem sehingga menghasilkan suatu keluaran yang sesuai dengan yang dibutuhkan.

 

1.7.    Metode Pengambilan Data

Melakukan Observasi

Mendapatkan data dengan cara melakukan penelitian langsung pada objek yang diteliti.

Wawancara

Mendapatkan data dengan cara bertanya pada pihak pengelola perpustakaan tentang proses pengolahan data anggota, petugas, buku, transaksi dan denda.

 

1.8.    Sistematika Penulisan

Dalam penyusunan laporan ini penulis menggunakan sistematika penulisan sebagai berikut :

BAB I    PENDAHULUAN

Pada bab ini penulis akan menjelaskan mengenai Latar Belakang

Masalah, Rumusan Masalah, Batasan Masalah, Tujuan Penelitian,Manfaat Penelitian, Motode Penelitian, Metode Pengambilan Data, dan Sistematika Penulisan Laporan.
