3. Algoritma merge sort dilakukan dengan prinsip divide and computer yang memiliki arti


Jawaban dan Penjelasan:

(koreksi: bukan divide and computer, melainkan divide and conquer)

Algoritma merge sort dilakukan dengan prinsip divide and conquer yang memiliki arti “bagi/pecah (menjadi beberapa bagian) dan taklukkan”, yang sesuai dengan prinsip dekomposisi, yaitu memecah masalah yang ada menjadi beberapa bagian kecil sehingga lebih mudah untuk diselesaikan.

Pada algoritma merge sort, list/data dibagi menjadi dua bagian secara rekursif hingga bagian terkecil, yaitu dua bagian yang hanya berisi satu elemen saja. Kemudian dari bagian-bagian terkecil, dibandingkan dan terjadi pertukaran posisi sesuai jenis pengurutan. Setelah itu, secara bertahap, semua bagian yang sudah terurut digabung kembali, dengan menyeleksi mulai dari elemen terkecil atau terbesar (tergantung jenis pengurutan: ascending atau descending) dan memasukkannya ke list/data gabungan. Hal ini dilakukan sampai semua bagian tergabung kembali menjadi satu bagian utuh yang sudah terurut.

Quick Sort

#include<stdio.h>  

void swap(int* a, int* b)  

{  

   int t = *a;  

   *a = *b;  

   *b = t;  

}    

/* Fungsi ini mengambil elemen terakhir sebagai pivot, menempatkan pivot di posisi yang tepat dalam array yang telah di sort, menempatkan nilai yang lebih kecil dari pivot ke bagian kiri pivot dan nilai yang lebih besar dari pivot ke bagian kanan pivot */

int partition (int arr[], int low, int high)  

{  

   int pivot = arr[high];    // pivot  

   int i = (low - 1);  // Index dari element yang lebih kecil

 

   for (int j = low; j <= high- 1; j++)  

   {  

       if (arr[j] <= pivot)  

       {  

           i++;    // increment dari indeks elemen yang lebih kecil

           swap(&arr[i], &arr[j]);  

       }  

   }  

   swap(&arr[i + 1], &arr[high]);  

   return (i + 1);  

}    

/* arr[] --> Array yang akan di sort,  

 low  --> Indeks awal,  

 high  --> Indeks akhir */

void quickSort(int arr[], int low, int high)  

{  

   if (low < high)  

   {  

       /* pi adalah indeks partisi */

       int pi = partition(arr, low, high);  

       quickSort(arr, low, pi - 1);  

       quickSort(arr, pi + 1, high);  

   }  

}

void merge(int arr[], int l, int m, int r)  

{  

   int i, j, k;  

   int n1 = m - l + 1;  

   int n2 =  r - m;

           j++;

Penjelasan:

Pola conquer pada algoritma Quick Sort adalah langkah di mana subarray yang lebih kecil dibagi menjadi sub-subarray yang lebih kecil atau elemen individual, dan kemudian diurutkan secara rekursif. Ini terjadi setelah proses pemisahan (divide) dalam algoritma Quick Sort. Dalam fase conquer, setiap subarray yang lebih kecil diurutkan secara terpisah menggunakan algoritma Quick Sort.

Berikut adalah langkah-langkah dan penjelasan pola conquer pada algoritma Quick Sort:

1. **Memecah Subarray:**

  Setelah pemisahan (divide) dilakukan, array awal terbagi menjadi dua bagian, yaitu subarray yang lebih kecil. Langkah conquer akan diterapkan pada setiap subarray ini.

2. **Urutkan Subarray Secara Rekursif:**

  Pada langkah conquer, setiap subarray yang lebih kecil akan diurutkan secara rekursif menggunakan algoritma Quick Sort. Proses ini melibatkan pemisahan (divide) kembali dan kemudian pengurutan subarray-subarray tersebut.

3. **Elemen Individual:**

  Pada titik tertentu dalam rekursi, subarray akan menjadi sangat kecil sehingga hanya terdiri dari satu atau dua elemen. Pada titik ini, subarray dianggap sudah terurut karena elemen individunya.

4. **Penggabungan:**

  Setelah semua subarray lebih kecil diurutkan, langkah conquer menggabungkan kembali subarray-subarray ini sehingga membentuk subarray yang lebih besar dan terurut. Namun, perhatikan bahwa penggabungan ini biasanya tidak memerlukan langkah tambahan dalam algoritma Quick Sort karena pengurutan telah terjadi secara in-place dalam array.

Pola conquer ini membantu menerapkan prinsip "divide and conquer" dalam algoritma Quick Sort. Dengan memecah masalah menjadi sub-masalah yang lebih kecil, algoritma ini secara efisien mengurutkan seluruh array dengan memanfaatkan rekursi untuk mengurutkan subarray-subarray yang lebih kecil. Setelah seluruh subarray terurut, array induk juga akan terurut karena sifat transitif pembandingan dalam algoritma Quick Sort.

29. Algoritma Bubble Sort disebut juga sebagai Sinking Sort. Mengapa demikian? Jelaskan!​


Jawaban dan Penjelasan:

Untuk menjelaskan mengapa algoritma bubble sort disebut juga sebagai sinking sort, kita gunakan prinsip kesetimbangan, yaitu “yang kecil naik dan yang besar turun”.

Kita bayangkan bahwa data direpresentasikan dalam bentuk susunan elemen data (seperti stack), di mana jenis pengurutan adalah menaik dari atas hingga paling bawah.

Dengan bubble sort, pada iterasi tertentu, jika suatu elemen data bernilai lebih dari elemen lain yang berada di bawahnya, maka terjadi pertukaran. Elemen yang lebih kecil naik 1 tingkat, dan elemen yang lebih besar turun 1 tingkat.

Anggap saja elemen terkecil berada pada posisi paling bawah dari susunan data belum terurut, dan elemen terbesar berada pada posisi paling atas dari susunan data tersebut. Maka, elemen terkecil akan naik secara bertahap hingga mencapai puncak susunan data terurut. Hal ini bisa dianalogikan dengan gelembung udara dalam air (bubble), yang akan naik hingga ke permukaan.

Sebaliknya, elemen terbesar akan turun secara bertahap hingga mencapai dasar (posisi terbawah) dari susunan data terurut. Hal ini dapat dianalogikan dengan sebuah batu (bukan batu apung) yang dimasukkan ke air. Batu tersebut akan turun (tenggelam, atau sinking) hingga dasar.

Jadi, dengan jenis pengurutan seperti disebutkan di atas, algoritma ini disebut bubble sort dengan memandang elemen terkecil; disebut sinking sort dengan memandang elemen terbesar.

38. jabarkan tentang pola divide algoritma quick sort!


Algoritma Quick Sort merupakan salah satu algoritma pengurutan yang bekerja dengan memanfaatkan pola divide and conquer. Di bawah ini adalah penjabaran mengenai pola divide dalam algoritma Quick Sort:

1. Tahap Awal:

- Pilih elemen pivot dari array yang akan diurutkan. Pivot dapat dipilih secara acak atau dengan menggunakan strategi tertentu, seperti memilih elemen tengah atau median dari tiga elemen.

- Bagi array menjadi dua bagian, yaitu array bagian kiri dan array bagian kanan, berdasarkan nilai pivot.

- Perbandingan dilakukan untuk memindahkan elemen-elemen yang lebih kecil daripada pivot ke bagian kiri dan elemen-elemen yang lebih besar ke bagian kanan.

2. Tahap Rekursif:

- Rekursif dilakukan pada kedua bagian array yang terbentuk (bagian kiri dan bagian kanan).

- Pada setiap rekursi, langkah-langkah yang sama dilakukan seperti tahap awal, yaitu memilih pivot dari bagian array yang lebih kecil dan membagi array menjadi dua bagian berdasarkan pivot.

3. Tahap Penggabungan:

- Setelah rekursi selesai, hasil pengurutan akan dikombinasikan kembali. Ini dilakukan dengan cara menggabungkan array bagian kiri, pivot, dan array bagian kanan dengan urutan yang tepat.

- Pivot ditempatkan pada posisi yang benar, sehingga semua elemen di bagian kiri pivot lebih kecil daripada pivot dan semua elemen di bagian kanan pivot lebih besar daripada pivot.

4. Tahap Berulang:

- Jika masih ada array yang perlu diurutkan atau dibagi lebih lanjut, tahap awal dan rekursif akan terus diulang pada setiap subarray yang terbentuk dengan pemilihan pivot baru di setiap langkahnya.

- Proses ini berlanjut hingga semua subarray terurut dan digabungkan secara tepat.

Melalui pola divide ini, Quick Sort memecah masalah pengurutan menjadi submasalah yang lebih kecil dan lebih mudah diatasi. Dengan memanfaatkan strategi pemilihan pivot yang efisien dan pengaturan elemen-elemen dengan benar pada setiap iterasi, algoritma Quick Sort dapat melakukan pengurutan dengan cepat dan efisien.
