3. Buatlah 3 contoh peristiwa dengan mengunakan Flowchart (Diagram Alir)dan Pseudocode


1. Memasak Telur

Kamu ingin memasak telur, ada variasi langkah flowchart yang bisa dilakukan:

Siapkan telur yang akan dimasak

Tentukan telurnya akan dimasak seperti apa

Kalau ingin dimasak orak-arik, maka:

3.1. Pecahkan telurnya ke dalam wadah

3.2. Kocok telur dan tambahkan bumbu sesuai selera

3.3. Panaskan penggorengan

3.4. Masukkan kocokan telur ke penggorengan yang sudah panas

3.5. Aduk telur sampai menjadi orak-arik

4. Kalau ingin dimasak ceplok, maka:

4.1. Panaskan penggorengan yang sudah diisi minyak

4.2. Pecahkan telur di penggorengan dan tunggu sampai matang

Kalau ingin dimasak rebus, maka: 5.1. Didihkan air dalam panci

5.2. Masukkan telur utuh ke dalamnya dan tunggu beberapa menit sampai isinya matang

Siap untuk dimakan

2. Menyalakan Komputer

Kamu akan menyalakan komputer, alur flowchart-nya:

Pencet tombol “on” di CPU

Apakah komputer sudah menyala normal?

Jika ya 3.1. Apakah muncul pesan error di layar?

3.1.1. Jika ya, lakukan troubleshooting

3.1.2. Jika tidak, komputer bisa langsung digunakan

Jika tidak4.1. Apakah lampu indikator menyala

4.1.1. Jika ya, pencet tombol “on” pada monitor

4.1.2. Jika tidak, cek sumber listrik untuk memastikan arus

4.2. Apakah masalah teratasi?

4.2.1. Jika ya, komputer bisa langsung digunakan

4.2.2. Jika tidak, silakan panggil teknisi untuk memperbaikinya

3. Memilah Sampah

Untuk merawat bumi, pemilahan sampah wajib dilakukan sebelum dibawa ke pembuangan akhir. Alur flowchart-nya kurang lebih:

Pilah sampah berdasarkan jenisnyaJenis pertama adalah sampah organikSampah organik dapat diolah menjadi pupukJenis kedua adalah sampah yang dapat digunakan kembaliSampah jenis ini dapat dimanfaatkan untuk hal lainSampah ketiga adalah sampah yang dapat didaur ulangSampah ini dapat didaur ulang menjadi barang lainJika tidak tergolong ke sampah organik, sampah yang dapat digunakan kembali, dan sampah yang dapat didaur ulang, baru dibuang ke tempat pembuangan akhir.

5. Apa persamaan flowchart dan data flow diagram


Persamaan flowchart dan data flow diagram adalah memiliki fungsi yang sama, yaitu untuk mempermudah kita dalam menggambarkan alur suatu proses. Pada data flow diagram sendiri simbol bangun datar yang tersedia untuk menjelaskan suatu proses tidak sebanyak simbol bangun datar pada flowchart.

Pembahasan

Setelah mengetahui apa persamaan dari flowchart dan data flow, maka selanjutnya kita akan menyebutkan perbedaan dari flowchart dan data flow diagram, beberapa perbedaan antara flowchart dan data flow diagram adalah sebagai berikut

Alur proses yang dijelaskan menggunakan flowchart harus digambarkan secara urut, sedangkan pada data flow bisa digambarkan secara serentak.Penggunaan simbol pada flowchart lebih banyak daripada data flow diagram.Pada flowchart tidak terdapat tingkatan jenis flowchart, pada data flow diagram terdapat tingkatan.

Untuk membuat flowchart dan data flow, kita bisa menggunakan aplikasi seperti microsoft word, microsoft visio.

33. Kak tolong besok udah di kumpulinBuatlah 3 contoh peristiwa dengan mengunakan Flowchart (Diagram Alir)dan Pseudocode. ​


Peristiwa dengan mengunakan Flowchart (Diagram Alir) dan Pseudocode contohnya adalah menghitung luas segitiga, keliling persegi,luas lingkaran

Pembahasan

Flowchart merupakan diagram yang menggambarkan alur kejadian suatu peristiwa, sedangkan pseudocode merupakan code program sementara yang mengikuti alur kejadian yang belum teruji kebenarannya.

pseudocode menghitung luas segitiga

start

Deklarasi

luas, alas, tinggi as integer

Proses

  Read(alas, tinggi)

  luas = 0.5 * alas * tinggi

  write (luas)

end

Pseudocode menghitung keliling persegi

start

Deklarasi

keliling, sisi as integer

Proses

  Read(sisi)

  keliling = 4 * sisi

  write (keliling)

end

Pseudocode menghitung luas lignkaran

start

Deklarasi

luas, r as integer

Proses

  Read(r)

  luas = 3.14 * r * r

  write (luas)

end

Diagram flowchart ini menggambarkan prinsip pemikiran komputasional yaitu input, proses, dan output. Input merupakan data yang dimasukkan ke dalam program, proses adalah pemrosesan data tersebut, dan output adalah hasil dari proses tersebut. Kemudian, program akan mengevaluasi apakah hasil proses sudah benar atau tidak. Jika hasil sudah benar, maka program selesai. Namun jika hasil belum benar, maka program akan mengulang proses dari awal.
