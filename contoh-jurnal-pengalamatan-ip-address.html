6. Berikut ini pernyataan yang benar, kecuali ... IP address 192.168.0.1/24 termasuk kelas C Opsi IP address 192.168.0.1/24 memiliki default subnetmask 255.255.255.0 IP address 192.168.0.1/24 memiliki alamat network 192.168.0.0 IP address 192.168.0.1/24 memiliki alamat broadcast 192.168.0.255​


Jawaban:

Pernyataan yang benar adalah: "IP address 192.168.0.1/24 termasuk kelas C".

Pernyataan yang salah adalah: "IP address 192.168.0.1/24 memiliki default subnetmask 255.255.255.0". Sebab, 255.255.255.0 adalah subnet mask default untuk kelas C, namun itu bukan berarti setiap alamat IP kelas C harus menggunakan subnet mask tersebut. Dalam hal ini, subnet mask 192.168.0.1/24 adalah 255.255.255.0, tetapi bisa saja menggunakan subnet mask yang berbeda.

Sementara itu, pernyataan "IP address 192.168.0.1/24 memiliki alamat network 192.168.0.0" dan "IP address 192.168.0.1/24 memiliki alamat broadcast 192.168.0.255" keduanya benar. Hal ini karena alamat jaringan (network address) selalu merupakan alamat IP pertama dalam subnet, sementara alamat broadcast selalu merupakan alamat IP terakhir dalam subnet. Dalam kasus ini, alamat jaringan adalah 192.168.0.0, sedangkan alamat broadcast adalah 192.168.0.255.

7. Apakah komputer yang terhubung ke jaringan boleh menggunakan alamat (ip address) yang sama?


Komputer mampu terhubung pada jaringan dan alamat yang sama adalah salah karena akan mengakibatkan konflik.

Pembasahan:

Protokol internet atau alamat IP adalah alamat pribadi Anda saat mengakses internet.  IP ini memungkinkan penyedia layanan Internet atau ISP Anda untuk menerima dan mengirim ulang aktivitas Internet yang dilakukan oleh seseorang. Selama Anda tidak berada di  jaringan yang sama, tidak ada masalah. Jika Anda berada di jaringan yang sama, konflik IP akan terjadi dan IP tersebut akan dianggap sebagai alamat yang buruk hingga salah satu atau keduanya mengubah IP. Tidak seperti alamat IP publik, alamat IP pribadi hanya berfungsi dalam jaringan area lokal (LAN). Menjadi lokal,  perangkat  dan tetangga  dapat menggunakan alamat IP pribadi yang sama. Artinya, alamat IP  adalah sekumpulan angka yang terkait dengan komputer, ponsel, atau perangkat "pintar" lainnya yang terhubung ke Internet. Nomor-nomor ini bervariasi dari perangkat ke perangkat dan digunakan untuk saling menghubungi.

16. tuliskan hubungan antara domain name system (DNS) dengan alamat ip ( IP address)​


Jawaban:

Pengertian IP address

Alamat IP (Internet Protocol Address atau sering disingkat IP) adalah deretan angka biner antar 32-bit sampai 128-bit yang dipakai sebagai alamat identifikasi untuk tiap komputer host dalam jaringan Internet. Panjang dari angka ini adalah 32-bit (untuk IPv4 atau IP versi 4), dan 128-bit (untuk IPv6 atau IP versi 6) yang menunjukkan alamat dari komputer tersebut pada jaringan Internet berbasis TCP/IP.

Pengertian Domain Name System (DNS)

Domain Name System (DNS) adalah distribute database system yang digunakan untuk pencarian nama komputer (name resolution) di jaringan yang mengunakan TCP/IP (Transmission Control Protocol/Internet Protocol). DNS biasa digunakan pada aplikasi yang terhubung ke Internet seperti web browser atau e-mail, dimana DNS membantu memetakan host name sebuah komputer ke IP address.

Penjelasan:

Hubungnnya adalah DNS dan IP adress adalah sama sama sebagai penamaan sistem, bedana jika DNS terdapat nama contohnya (namadomain.com) sedangkan ip dari namadomain.com (192.178.0.1)

19. Tuliskan cara menentukan pengalamatan ip address


Penjelasan :

IP Address (Internet Protocol) merupakan deretan angka biner antara 32-bit sampai 128-bit yang digunakan sebagai alamat identifikasi tiap host komputer dalam jaringan Internet.

Cara menentukan pengalamatan IP Address

1. Addressing, IP Address mengidentifikasi secara detail untuk di setiap host jaringan, sehingga dapat menjamin data yang dikirim tertuju pada alamat yang benar.

2. Routing, Ini merupakan pengaturan gateway yang bertujuan untuk mengirim data kepada jaringan dimana host tujuan berada.

3. Multiplexing, Ini merupakan pengaturan nomor port dan protokol yang mengirim data pada modul software yang benar di dalam host jaringan.
